# Paid Traffic Output Quality Checklist

**Checklist ID:** TRAFFIC-CL-001
**Referenced by:** tasks/review.md
**Purpose:** Validate paid traffic deliverables for quality before delivery to user.

[[LLM: INITIALIZATION INSTRUCTIONS

This checklist validates paid traffic output specifically.

EXECUTION APPROACH:
1. For each category, verify every item against the deliverable
2. Mark items as [x] Pass, [ ] Fail, [N/A] Not Applicable
3. CRITICAL items block delivery; non-critical items are advisory

CRITICAL items are marked with (CRITICAL) suffix.]]

---

## 1. Targeting & Audience

- [ ] Target audience is defined with specificity: demographics, interests, behaviors (CRITICAL)
- [ ] Custom/lookalike audiences are considered where appropriate
- [ ] Exclusion audiences defined to avoid wasted spend
- [ ] Audience size is appropriate for the budget and objective
- [ ] Geo-targeting is specified and justified

## 2. Campaign Structure

- [ ] Campaign objective matches the business goal (CRITICAL)
- [ ] Campaign > Ad Set > Ad hierarchy is logically organized
- [ ] A/B test variations are built in where appropriate
- [ ] Funnel stage alignment: awareness, consideration, conversion
- [ ] Retargeting strategy defined for each funnel stage

## 3. Creative & Ad Copy

- [ ] Creative follows platform-specific best practices (aspect ratio, text limits, format) (CRITICAL)
- [ ] Hook is present in the first 3 seconds (video) or first line (text)
- [ ] CTA is clear, specific, and matches the landing page
- [ ] Ad copy complies with platform advertising policies (no prohibited claims)
- [ ] Multiple creative variations provided for testing

## 4. Budget & Bidding

- [ ] Budget allocation is justified with rationale (CRITICAL)
- [ ] Daily vs lifetime budget choice is explained
- [ ] Bid strategy matches the campaign objective
- [ ] Budget is sufficient for the audience size and learning phase
- [ ] Scaling plan defined: when and how to increase spend

## 5. KPIs & Measurement

- [ ] Primary KPI is defined and measurable: CPA, ROAS, CPL, CTR (CRITICAL)
- [ ] Tracking pixels/conversions are specified for implementation
- [ ] Attribution model is stated (last-click, first-click, multi-touch)
- [ ] Benchmark values provided for comparison
- [ ] Reporting cadence and optimization checkpoints defined

## 6. Landing Page Alignment

- [ ] Ad message matches landing page message (scent continuity)
- [ ] Landing page load speed considered
- [ ] Conversion mechanism on landing page is defined
- [ ] Mobile experience verified or flagged for verification

---

## PASS/FAIL Criteria

**PASS:** All CRITICAL items [x] and fewer than 3 non-critical failures.
**REVISE:** All CRITICAL items [x] but 3+ non-critical failures.
**FAIL:** Any CRITICAL item unchecked.
